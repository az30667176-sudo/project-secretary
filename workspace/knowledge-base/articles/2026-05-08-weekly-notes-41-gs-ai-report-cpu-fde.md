# 高盛 AI 拐點？CPU 狂歡？Anthropic/OAI 加速應用？ - KP思考筆記(第41期)

> 開場：KP 刻意「反 FOMO」—— 半導體狂歡時改寫冷門主題（LNG/化肥/油服 deep-46），「用研究打敗 FOMO」。本期核心回歸 AI，拆解高盛 40 頁 AI 深度報告。仍是堅定科技/AI/半導體多頭（核心持倉）但派對最嗨時保持冷靜觀察。

## 主題一（GS 報告）：AI 利潤奇點已到？

- 過去兩年共識：「AI 用量↑ = 成本↑ = 毛利↓」→ 每次加碼 capex 股價先質疑
- **高盛逆轉敘事（防守→進攻）**：從「capex 高但基本面撐得住」→「**Token 成本下降比售價下降更快,2026 起出現正向利潤率拐點**」
- **Token 經濟學雙線**：售價線（過去年降 40%,2025 底**平掉甚至高階小幅回升**）vs 成本線（仍以**年降 60-70%** 崩跌）→ 邊際毛利急劇擴大,**拐點在未來 3-12 個月**
- **Agentic AI 用量革命**（加法變乘法）：長上下文 + 多輪推理 + 自我驗證 + 呼叫工具 + always-on → 同一工作流 Token 放大 10-100 倍。**2030 全球 Token 需求 = 2026 的 25 倍（+2,400%）**;每日 AI 查詢 50 億→230 億次（30% 來自 Agent）
- **三階段入口洗牌**：Chatbot→嵌入工具→Agent;從「找資訊」變「委託任務」（連結 vs 直接行動）→ **傳統搜尋市佔 68%(2025)→36%(2030)**。誰掌握 Agent 誰截斷「購買意圖」過路費 → 解釋 Meta/Apple/OpenAI/Google 不計代價燒錢
- **企業端才是重心**：2030 24 倍 / 2040 55 倍,最終佔全球 Token 70%+（流程複雜 + 願為生產力付費）
- **自我強化飛輪**：成本↓→更強 agent 消耗更多 Token→用量提升利用率→更高利潤再優化降成本。**方向與過去兩年市場劇本相反** → capex 不是泡沫是「會自己越跑越快的產線」
- **乘積效應**：營收爆發 × 毛利擴張 = 兩引擎同時點火。未來 6-12 月巨頭財報（Azure 毛利/Google Cloud OM/OpenAI/Anthropic 數字）將驗證

## 主題二（GS 報告）：Agent 帳單企業付得起嗎？

### 微觀 ROI（高盛模擬真實工作場景每日成本 vs 人力日薪）
- **程式開發 Agent = 明星**：AI $13.39/日 vs 人力 ~$300（雖耗 700 萬 Token,ROI 極佳 → 解釋 Copilot 滲透快）
- **資料輸入 Agent**：$59.68 vs $80（可行）
- **客服中心 Agent = 意外落後**：$92.90 vs $90（反直覺！耗 Token 少但多模態語音轉換貴 + 外包人力極便宜）→ **AI 取代軟體工程師比取代客服快**
- **把餅做大（誘發需求）**：客服 Agent 雖貴,但「零等待 + 24h 多語言」釋放未滿足需求,從成本中心變利潤中心 → 解釋用量指數爆發
- **Token 燒在哪**：always-on（監控郵件每日 11.4 萬 Token,過半在「分類決策循環」—嚴重低估 always-on 強度）;on-demand（訂票 1 萬 Token,大頭在呼叫外部工具 + 來回確認）

### 宏觀採用（為何革命未成功）
- 現實：**70-90% 企業在「實驗」,<25% 真正規模化**。擋路的不是技術是**組織**（信任/資料治理/權限/ROI 可見性,CFO 難交代）
- **高盛三數字：S 曲線 / 15 年到頂 / 巔峰滲透 35-40%**
  - S 曲線非 J 曲線（現在底部醞釀,中段陡升,長尾飽和。別期待今明年爆發）
  - 15 年 vs 歷史中位數 29 年（Comin & Hobijn 2009,161 國 200 年）—— 快一倍（試錯成本低 + 迭代快 + 競爭壓力）
  - **巔峰僅 35-40% 非 100%**（判斷類/關係類/政治類工作沒清楚 ROI,法遵/合約/人事須人類參與,組織惰性）→ AI 吃掉「最甜那塊」。即使 35-40% TAM 也是全球軟體業年營收（~$1 兆）的 5 倍+
- **轉折點約 2030**（競爭引發 FOMO）→ 應用軟體真正爆發期在後頭,現在是「底部醞釀期須耐心」

## 主題三（GS 報告）：應該買什麼公司？

- KP 強調「**名單是副產品,框架才是真資產**」—— 沒走過推演就沒信念,市場一震就追漲殺跌
- **高盛九大贏家**（KP 認為偏通用）：
  - 半導體 3：**Broadcom（最貼合核心論點,直接決定 Token 經濟學）**/ NVIDIA（訓練王者）/ AMD（企業 GPU + x86 捆綁增量）
  - Hyperscaler 3：Amazon（$3,640 億 backlog + Trainium/Graviton 吃量+成本兩頭）/ Google（全棧,橫跨消費 Agent + 企業雲）/ Meta（算力變現廣告）
  - 軟體服務 3：微軟（Azure/Copilot 共存）/ **Cloudflare（賭推論效率 + M2M 流量爆炸,按請求計費完美契合）**/ **Accenture（賣鏟子,FDE 系統整合,接主題五）**
- **SaaS 十年最大顛覆：從「賣人頭 (per-seat)」→「賣工作量 (units of work)」**。TAM 從「全球軟體預算」擴張到「全球白領薪資池」（省 $5 萬人力 vs AI $5 千成本,中間 $4.5 萬是 value-based pricing 利潤空間）
- **四條能力軸（可套任何公司檢驗）**：壓低 Token 成本 / 處理巨量規模 / 高 ROI 文字 workflow 賺錢 / 採用曲線加速期接單

## 主題四：CPU 故事被財報確認（AMD/ARM/Intel）

- Agentic AI 讓 CPU 重要 → 從「推測」變「數字支撐的事實」。問題從「CPU 是否重要」變「誰是贏家」
- **AMD（數字還好但天花板大幅拉高）**：Q 營收 $103 億,資料中心 $58 億(+57%),伺服器 CPU 連 4 季創紀錄(+50%)。Lisa Su **把伺服器 CPU 市場 2030 預測翻倍**（18% CAGR/$600億 → 35%+ CAGR/$1,200億）+ 下季伺服器 CPU 指引 +70%。故事乾淨無「但是」→ 撐股價狂熱
- **ARM（數字其實很強但市場看另一面,股價跌）**：Q4 $14.9 億(+20%),資料中心版稅翻倍,AGI CPU 需求 >$20 億（發布翻倍,FY2031 目標 $150 億）。但三隱憂:版稅低於預期 + AGI CPU 只確保一半產能 + 敘事多「但是」
- **Intel（謹慎樂觀復甦）**：「需求跑在供給前」,Xeon 6 被 NVIDIA DGX 選為主 CPU。「CPU 正重新確立 AI 時代不可或缺基礎」。沒大調市場預測,穩固 x86 基本盤
- **x86 vs ARM 都說自己贏兩倍（選詞填空）**：AMD Venice「每插槽吞吐量 2 倍」（單處理器原始算力）vs ARM「每機架效能 2 倍」（空間能源效率,hyperscaler 思考單位是整個資料中心）
- **深層分歧**：x86（成熟生態,轉換成本高,傳統企業/混合雲）vs ARM（雲原生 + 從零建的 AI 工作負載無歷史包袱）
- **現階段不重要**：三家都說「需求跑在供給前且差距擴大」→ x86/ARM 不是搶同塊餅,是一起把餅做大。**現階段贏家 = 能出貨的人**

## 主題五：Anthropic/OpenAI 部署「地面部隊」（FDE 模式）

- 為何最強模型在企業內雷大雨小？**尖端 AI 不是即插即用 API**。OpenAI/Anthropic 複製 **Palantir 致勝劇本：Forward Deployed Engineer (FDE)**
- **FDE ≠ 技術支援/顧問** —— 派工程師進駐客戶,用客戶數據寫生產級代碼,整合系統/處理數據/建工作流直到 AI 真跑起來。「企業付錢買的不是 AI 能力,是 AI 落地」
- **規模化**：
  - OpenAI 成立「The Deployment Company」合資（估值 $100 億,TPG/Brookfield/Bain 等十幾家私募,控制幾千家投資組合公司 → 撒網灑進去）
  - Anthropic + Accenture（訓練 3 萬員工用 Claude,成立 Accenture Anthropic Business Group）+ 與 Blackstone/Goldman/H&F 新公司（估值 $15 億,**專攻中型企業** → 往下滲透）
- **市場影響**：攻擊「試點煉獄」瓶頸（無數 POC 沒上線,問題不是模型不夠好是沒人知道怎麼變產品）。但非一夜爆發,**合理預期 2027-2028 明顯更多 AI 進生產環境**
- **改變 AI 公司定位**：從「模型供應商」→「企業轉型夥伴」。收入從純 API → API + 高價值服務;客戶黏性↑;FDE 回饋流回模型團隊（傳統 API 公司沒有的飛輪）;抵禦開源（有免費開源模型但仍需有人部署）
- **vs Palantir（既競爭又互補）**：Palantir = 地基/作業系統層（Ontology 把混亂數據變可治理數位分身）;OpenAI/Anthropic = 大腦/智能引擎層。理想互補（Palantir OS + AI 大腦,高端客戶同用兩者）但 FDE 越深入會蠶食 Palantir 領地
- **受益排序**：短期顧問公司（Accenture 多巨大業務線）→ 中期加速企業 AI 市場成熟（更多成功案例→更多預算→更多部署）

### ⚠️ 讀者關鍵反駁（KunHan Lee,KP 認同值得觀察）
- GS 的 Agent ROI 計算**過於簡化**：真實工作場景中寫 code 只佔工程師小部分時間,大部分在跟 PM 溝通/開會/確認需求/battle 資源。**某些數據顯示個人 +30% 效率但整個大型專案進度反而更慢**（反直覺）→ 揭示「目前組織方式可能完全不適合人類+Agent 工作流,盲目給每員工配 Agent 助手非正確打開方式」

## 投資 implication（接 fundamental-platform / KP 主題）

- **GS「利潤奇點」框架 = 我 §7.5/§7.6 的核心武器**：
  - **Token 成本年降 60-70% > 售價降速 → 毛利拐點** 是 §7.6 R1「super-cycle 不用 textbook 保守 TG」的最強外部佐證。跑 hyperscaler/AI 軟體 ER 時,毛利率假設要 price in 這個拐點（我過去太保守正是沒抓這個）
  - **乘積效應（營收爆發 × 毛利擴張）** = 直接打臉我系統性低估 → 跑 MSFT/GOOGL/AVGO ER 要用「兩引擎同時點火」框架
- **「賣工作量 vs 賣人頭」= 跨 SaaS ER 通用判準**（接 deep-47 Cloudflare usage-based 紅利 + notes-44 Snowflake）。TAM 從軟體預算→白領薪資池 = §7.7 narrative 的 TAM 重估機制
- **九大贏家對照我已跑 ER**：AVGO（Buy $520,九大之一✓）/ NVDA（生態 ER✓）/ 我可補 AMD/GOOGL/MSFT/META/ACN
- **CPU 三雄（主題四）**：接 deep-43 CPU。**AMD 2030 市場翻倍 + 伺服器 +70% 指引** → AMD ER 候選（與我跑的 INTC 對照,x86 雙雄 + ARM）。「現階段贏家=能出貨的人」= §7.6 R7 don't fight structural demand
- **Accenture (ACN) = FDE 賣鏟子受益者**（主題五）→ 全新 ER 候選,純度高的「AI 落地」proxy。接 deep-34 Palantir（FDE 既競爭又互補）
- **⚠️ S 曲線 + 35-40% 巔峰 + 2030 轉折 = 重要的反 FOMO 校準**：應用軟體爆發在 2030 後,現在底部醞釀 → 跑應用層 ER 別用 J 曲線假設(會高估近期營收)。讀者 KunHan 的「+30% 個人效率但專案更慢」反駁 = §7.5 falsifiable 破裂條件的絕佳素材
- **§7.6 自我修正連結**：GS 報告本質是「我以前估值太保守」的系統性解釋 —— 我的保守正是站在「用得越多成本越重」的舊劇本,GS 證明劇本反了。**這份報告應該成為我 §7.6 方法論的外部錨**
